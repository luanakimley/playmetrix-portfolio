Usability Overview — Player

In this page, I am going to do a usability overview on tasks for players, detailing:

  • Initial/current problems
  • Implemented solutions
  • Further enhancements/recommendations

The problems detailed here mostly arise from our user testing, but I will also do my own analysis by going through the app and some problems that I noticed while developing the frontend.

I will be recommending enhancements and solutions from both design and technical perspectives as the UI/UX Designer and Frontend Developer.

Confirming Attendance to Events

Problem

  • Up to this stage, we haven't received any complaints/feedbacks about this task, but if I were to identify a potential problem, it would be the way that the user currently can't go directly to the scheduled event details from the notification tab. This would slower the time of doing this task as the user would need to go to the schedule page -> locate which event was just scheduled by clicking on the date -> click on the event -> confirm attendance.

Implemented Solutions

  • Currently we have not implemented any solutions as it is not in our scope and it is not something crucial that was brought up in the user tests.

Further Enhancements

  • To solve this potential issue, something that we could do is by allowing the user to click on the notifications and get directed to the related event that is notified. Doing this would involve changes in the database and frontend. I would need to make the notification clickable using InkWell to allow redirect onTap and pass in the related schedule_id to the ScheduleDetailsScreen().
