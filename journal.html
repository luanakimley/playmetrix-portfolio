Sprint #6 (Reflection 2.0) - 15/11/2023-20/11/2023

Goals

  • Finalise our prototype based on more user feedback

Progress

After discussing with the panel regarding our feedbacks for T-CA1 and conducting user testings, we decided to dedicate another sprint for reflecting on the features of our app, narrowing down our scope, and focusing on the main features of our application. We also had meetings with potential external collaborators, Louth GAA and DkIT Sports.

Louth GAA wanted the application to be used for academy players, focusing on the communication and organisation between different components of the club. We had a discussion as a team and decided that it is not feasible as first, we would need to think of the ethical, data privacy, and other issues related to making application for children under 18. Another thing they wanted was a communication channel for them to be able to communicate in-app to replace other communication applications, such as WhatsApp. This would take a long time to develop and they wanted the app to be fully functional and usable by January/February 2024. We felt like this is not feasible and way out of scope so we did not go ahead with this collaboration.

The next meeting we had was with the DkIT Sports officers and it went quite well as they were interested in it and gave some valuable inputs, including an app that we could look into. They also said that they were going to help us with some user testing for January with potential users of the app. They were also happy to help throughout the year if we want some inputs or feedbacks on our application, which would be valuable to us as a team to get feedbacks from experts and experienced people in this field and make our app more useful and helpful for our users.

As the UI/UX Designer developing the prototype, these meetings and all the user inputs and feedbacks are quite overwhelming to me as I would need to keep changing our prototype to adhere to all these inputs given. A thing we did not think of and we just realised after the meetings is that we did not think of a way to get the statistics of the total minutes played for a player. We then had to think of a way to do this within our app and came up with this idea of having a form that would keep track of the time the match starts, ends, and the start and end time of each players involved in the match, as seen here:

Reflection

I feel like this sprint was a crucial week for us as a team as we have to finalise our concept as we are going to start development in the next sprint, so we have to actually know and understand the full scope of our project, along with its features and what we want to focus on. In my opinion, we did well and we know what we're going to be doing for the next release, which is a good start before the next sprint.

Sprint #10 (UI Structure Development 4.0) - 2/1/2024-15/1/2024

Goals

Finish off the UI structure development:

  • Add announcements screen

Personalise the flow for each user type of our application, including:

  • Manager flow
  • Coach flow
  • Physio flow
  • Player flow

Progress

During this sprint, my work mostly consists of personalising the flow of each user, ensuring that each of them has proper access levels to the features. So for example a coach and a manager can view the same "players" screen, but a manager will be able to see the "add player" button while the coach will not be able to see it.

I had to do research on how to know which user role is currently logged in and what I found was that I had to set a global state for the app using the Riverpod library in Flutter. Using this library, I can have providers that I could set during sign up and log in to know which particular user in logged in and save that in the provider. It took quite awhile to set up but once it was set, implementing it was handy enough to do.

Reflection

Setting up Riverpod was quite a hard thing to do, I feel like I should have done more research on this initially before starting any development, but it all worked out great at the end so I would keep this as a note for my future projects. A big accomplishment in the frontend is also achieved in this sprint as all the pages structure has been developed. My target for the next sprints is to focus on the functionality of the screens by connecting to the backend. It is essential to also test that it is connected properly without any bugs and is showing the data and doing proper CRUD operations.

Goals

  • Develop recording duration functionality

  • Develop optional 2FA screen and functionality

Progress

A major thing done during this sprint is the recording duration functionality. I used a Flutter package called duration_picker to allow the manager to add a duration for each player for each match. Other than that, I also made minor pages such as change password and optional 2FA to enhance the security of our app and further improve its usability.

Other bugs included wrong states, validations, and fetches.

Overall after this sprint we felt more confident in approaching our second release deadline. Sprint #18 (Final Bug Fixing, Planning User Tests) - 10/4/2024-15/4/2024

Goals

  • Fix prioritised frontend bugs
  • Research and plan user tests for showcase features and recording duration

Progress

Most of the frontend fixes I did during this sprint revolves around implementing deletes in the UI, such as deleting player injury by a physio and deleting scheduled event and announcements by a manager. For each delete, I made sure to add validations to prevent the user from accidentally deleting things through a misclick.

Another thing that was brought up was how there is a potential security breach on the notifications of player injury, as it revealed too much information on the player's injury. So to fix that I modified the notification injury to only show that the player was injured, and did not show any detailed information on the injury.

In addition to that, I also helped Shakira in planning the user testing for the showcase feature and record duration, which are the new features that we added in the last release. A more detailed plan of the user test can be seen in the "Usability Overview" section of my portfolio. While planning the user test, I mostly focused on questions that could give insights on the usability of the features and more context on where and when the user would more likely do the task.

Reflection

After a long period of development, the panel advised us to finally freeze development and focus more on testing and the expo after this sprint. Overall we already have a complete application, but bugs are something that is inevitable for all applications, same with our ones, but we have fixed most of the major breaking bugs, so our app is now relatively stable and demoable for the expo.
