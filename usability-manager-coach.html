Usability Overview — Manager & Coach

In this page, I am going to do a usability overview on tasks for managers and coaches, detailing:

  • Initial/current problems
  • Implemented solutions
  • Further enhancements/recommendations

The problems detailed here mostly arise from our user testing, but I will also do my own analysis by going through the app and some problems that I noticed while developing the frontend.

I will be recommending enhancements and solutions from both design and technical perspectives as the UI/UX Designer and Frontend Developer.

Adding Players to the Team

Problem

  • A problem that arises during user testing with DkIT Sports is that the manager is having difficulties keeping track of the players in the team, having to add each of their emails and details one-by-one.

Implemented Solutions

  • Currently, we do not have a solution implemented yet for this problem, as we have time constraints that does not allow us to dive deeper into this problem, so currently a solution that we can propose to the user is to make some kind of Google form for the manager/coach to ask the players to fill in their email, and then Google Form can generate a spreadsheet containing all the player's email data so the manager/coach can look at it while entering the player details in our app.

Further Enhancements

  • Importing data through spreadsheets - This can be used in combination with the Google Form approach. We could provide the manager a template for them to distribute to their players to fill in and we can implement the importing of a spreadsheet that contains the player details into our database through the backend.
Recording Statistics (Duration, Matched Played)

Problem

  • Initially, the manager needs to manually input the numbers for each player using a +, -, or entering the total duration played by a player from their edit profile menu. This makes it hard for the manager to input and keep track of the duration and matches played as they will need to do manual calculations.
    • We had a comprehensive analysis on the implementation of this feature during our Human Factors in Design module, where we did a probe on the human error and performance, cognitive, and ergonomic problems the user might face while doing this task, and a lot of problems did come up, some of them including the user being cognitively overloaded and how the conditions would not support the user to do such a hard task.

Implemented Solutions

  • Automatic calculation from duration inputed for each match - In order to reduce the cognitive load of the manager/coach and improve the usability of the app, instead of having them manually inputting the total duration played and matches started, we implemented this feature which automates the calculation of those statistics by having the manager input the duration of the player for each match.

Further Enhancements

  • From my UI redesign from my Human Factors in Design module CA, I came up with a solution to help tackle this problem, which is an implementation of a "Match Mode" that the manager can use during a match to further automate the calculation of the duration. The Match Mode will feature a simplified UI mode which will only feature specific functionalities that the manager/physio/coach will use during a match. A button/switch will be implemented in the UI so that the manager/coach can click/toggle on it to switch to this mode when a match is starting.

    • The implementation of this mode would also be useful in calculating the matches started and matches off the bench statistics we have, which currently still needs to be added manually by the manager/coach. This mode can keep track of which players are starting on a particular match, as well as who is being sent in as a substitute. Changing Line Up Status

Problem

  • It is highlighted after the user testing with DkIT sports that the manager is having a hard time selecting match line up, moving players from starting, sub, or reserves. Currently, the only way to do this change is by going to the players profile -> edit player -> change lineup status.

Implemented Solutions

  • Currently we haven't implemented any solutions for this problem as it is out of our scope and we didn’t have enough time.

Further Enhancements

  • Allowing changes to lineup status in match line up screen - Doing this will definitely make it easier for the manager/coach to change the lineup status as this is the page where they can see the current line up for the match. Having a drag and drop feature to move players around is also something that could be considered.
